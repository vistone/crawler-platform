## 简介

本项目采用模块化的配置管理架构，每个组件都有独立的配置文件和加载机制。配置系统主要包含以下特点：

- **模块化设计**：客户端和服务器分别拥有独立的配置系统
- **简洁高效**：专注于核心功能的配置，避免过度复杂的配置层次
- **类型安全**：使用Go结构体和TOML标签确保配置类型安全
- **易于维护**：每个模块的配置独立管理，便于维护和扩展

## 常见配置错误排查

### 配置文件解析错误

#### 错误现象
```
解析配置文件失败: toml: line 15: unexpected character: '!'
```

#### 排查步骤
1. 检查TOML语法是否正确
2. 确认引号使用是否匹配
3. 验证数值类型是否正确

#### 解决方案
```toml
# 错误：特殊字符未转义
description = "This is a test! Important."

# 正确：使用双引号
description = "This is a test! Important."
```

### 配置项类型错误

#### 错误现象
```
解析失败: cannot unmarshal string into Go struct field ProtocolConfig.type of type string
```

#### 排查步骤
1. 检查配置项的数据类型
2. 确认TOML值格式正确
3. 验证结构体标签匹配

#### 解决方案
```toml
# 错误：数值格式的字符串
type = 100

# 正确：字符串格式
type = "grpc"
```

### 配置验证失败

#### 错误现象
```
无效的协议类型: invalid (支持: grpc, tuic, both)
```

#### 排查步骤
1. 检查配置值是否符合约束条件
2. 确认配置项是否有默认值
3. 验证配置加载顺序

#### 解决方案
```go
// 确保配置值有效
if config.Protocol.Type != "grpc" && config.Protocol.Type != "tuic" && config.Protocol.Type != "both" {
    config.Protocol.Type = "both" // 设置合理默认值
}
```

### 配置文件不存在

#### 错误现象
```
配置文件不存在: cmd/grpcclient/config.toml
```

#### 排查步骤
1. 检查配置文件是否存在
2. 确认文件路径是否正确
3. 验证文件读取权限

#### 解决方案
```go
// 检查文件存在性
if _, err := os.Stat(path); os.IsNotExist(err) {
    fmt.Println("警告：配置文件不存在，使用默认配置")
    return defaultConfig()
}
```

### 日志配置问题

#### 错误现象
```
无法打开日志文件: permission denied
```

#### 排查步骤
1. 检查文件权限设置
2. 确认目录存在且可写
3. 验证路径格式正确

#### 解决方案

logger, err := NewFileLogger("./logs/app.log", true, true, true, true)
